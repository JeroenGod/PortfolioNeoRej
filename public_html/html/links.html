responsive web design

Smartphones en tablets zijn al een tijdje in opmars. Door hun afmetingen zijn ze zeer handig om mee te nemen, maar geven bepaalde websites quasi onleesbaar weer of de lay-out is volledig door elkaar gehaald. De oplossing was om een aparte mobiele site ernaast te maken, wat veel tijd in beslag neemt. Nu is de nieuwe trend om éénzelde website op maat weer te geven voor zoveel mogelijk platforms. Dit fenomeen noemt men Responsive Web Design. Voorbeelden hiervan vind je bij The Guardian en BBC News.

Ik heb getracht mijn website ook zo responsive mogelijk te maken. Door middel van CSS3 media queries en common techniques in RWD heb ik mijn stylesheet opgedeeld in 5 stukken: voor smartphones, tablets in staande en in liggende oriëntatie, netbooks en desktops. Mijn stylesheet is dan ook vrij lang geworden.

Niet alleen in de afmetingen heb je verschillen, browsers geven een website elk op hun eigen manier weer. Verscheidene eigenschappen van CSS3 worden niet ondersteund in bepaalde browsers. Microsoft's Internet Explorer is daarin wel een heel speciaal geval. De oudere versies ondersteunen bijna niets. Ik heb mijn site uitvoerig getest in Google Chrome, Mozilla Firefox en Android. Dankzij deze handige tools kun je stylesheets ontwerpen die compatibel zijn voor alle browsers. Voor alle duidelijkheid: ik heb ze zelf (nog) niet gebruikt.

android

Om mijn website op een Android smartphone te testen heb ik verschillende mogelijkheden geëxploreerd. Eerst uploade ik mijn site via Filezilla naar de server van de school om ze daarna te openen in mijn Samsung Galaxy gsm. Redelijk omslachtig. Na veel vijven en zessen vond ik uiteindelijk een tutorial om via een Android Browser Emulator mijn aanpassingen in Netbeans sneller te testen.

Download en installeer Android SDK Tool for Windows om met een bestaande IDE (editor) te werken; start 'SDK Manager', selecteer je Android platform(s) naar keuze en installeer ze; start 'AVD Manager', configureer je 'Android Virtual Device', start hem.
In Netbeans > Tools > Options > Misc > Mobile Platforms: typ hier je Android SDK path (C:\Program Files\Android\android-sdk) - run project in Android Emulator mode.

Nog een paar andere methodes om je project op je PC in een virtuele Android omgeving te testen:

Internet Explorer

Met Internet Explorer heb ik de meeste problemen ondervonden: tot en met IE8 is er geen support voor CSS3. Naar ik gelezen heb op One Extra Pixel kan je dit omzeilen door bijvoorbeeld een Javascript document toe te voegen. In latere versies worden meer toepassingen ondersteund door de vendor-prefix "-ms-" toe te voegen aan de CSS3 property. In de lijst van CSS compatibiliteit in Internet Explorer kun je verifiëren wat juist in welke versie ondersteund wordt.
